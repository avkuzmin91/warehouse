# # МОДУЛЬ: СПРАВОЧНИКИ

---

# 1. Цель
Реализовать систему справочников:

- Клиенты
- Цвета
- Размеры
- Товары

Все справочники доступны только администратору и используются в других модулях системы.

---

# 2. Общие требования

## 2.1 Доступ
- создание, редактирование, удаление доступно только роли **admin**
- остальные роли не имеют доступа к справочникам

---

## 2.2 Общие поля (для всех справочников)

- id: UUID
- name (Название): string, обязательное
- is_active (Не актуален): boolean
- created_at (Дата создания): datetime
- creator (Создал): выбор из списка Users

---

## 2.3 Бизнес-правила

- поле "Название" обязательно
- "Не актуален" = true означает, что запись не используется
- удаление и сохранение редактирования требуют подтверждения

---

## 2.4 Подтверждение удаления

При попытке удаления показывать сообщение:

"Данные в карточках могут быть утеряны"

Кнопки:
- Да → удалить
- Нет → отмена

---

## 2.5 Подтверждение редактирования

При попытке сохранении изменений показывать сообщение:

"Данные в карточках будут заменены"

Кнопки:

- Да →  сохранить изменения
- Нет → отмена

# 6. Справочник товаров

## Поля:

- Название товара (обязательное)
- Тип (enum: "одежда", "техника")
- Артикул товара (обязательное, уникальное)
- Поставщик (строка)
- Фото товара (image upload: jpg, png)
- Не актуален (boolean)

---

## API:

- GET /products
- POST /products
- PATCH /products/{id}
- DELETE /products/{id}

---

## Требования к фото:

- хранение пути (image_url) в базе

# 8. Ограничения UI

- не использовать редактирование в таблице
- не использовать модальные окна для форм
- использовать отдельные страницы:
  - список
  - создание
  - редактирование

---

# 9. Безопасность

- все API защищены (только admin)
- frontend не является источником прав доступа
- backend проверяет роль пользователя

---

# 10. Проверка

## Сценарий:

1. Войти под admin
2. Перейти в "Справочники"
3. Открыть любой справочник
4. Создать элемент
5. Открыть элемент
6. Отредактировать
7. Удалить → проверить подтверждение

---

## Проверка прав:

- зайти под user или manager
- попытаться открыть /dictionaries

Ожидается:
- отказ доступа или редирект
