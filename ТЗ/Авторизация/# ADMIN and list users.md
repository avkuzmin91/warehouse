\# AUTH EXTENSION: ROLE ADMIN + USERS LIST

# \# 1. Цель

Добавить новую роль **admin**, которая имеет доступ к просмотру всех пользователей системы.

\---

# \# 2. Роли системы

\## user (default)
\- регистрация
\- вход
\- доступ к /dashboard
\- НЕ имеет доступа к списку пользователей

\---

\## admin
\- вход в систему
\- доступ к /dashboard
\- доступ к списку всех пользователей
\- может вызывать endpoint /users

\---

# \# 3. Изменения в Data Model

\## User

Добавить поле:

\- role: string

Допустимые значения:
\- "user"
\- "admin"

\---

\## Правила:
\- по умолчанию при регистрации создаётся user
\- admin НЕ может быть создан через публичную регистрацию
\- admin создаётся:
  \- либо вручную в базе
  \- либо через seed (при старте приложения)

\---

# \# 4. Backend изменения (FastAPI)

# ## 5. Бизнес-правила

- обычный пользователь НЕ может получить список пользователей
- admin видит ВСЕХ пользователей
- пароль НЕ возвращается
- role всегда возвращается

# ## 6. Seed администратора

При запуске backend:

- если нет пользователя с role=admin
   → создать:

email: admin@example.com
 password: admin123

(пароль обязательно захешировать)
