# **UNIVERSAL: ПРОСТЫЕ СПРАВОЧНИКИ (2 ПОЛЯ)**

------

# 1. Цель

Реализовать единый стандарт для справочников с простой структурой:

- название
- статус активности

Применяется для:

- Типы товаров
- Цвета
- Клиенты
- Поставщики
- Размеры

------

# 2. Data Model (универсальная)

```
{
  id: string
  name: string
  is_active: boolean

  created_at: datetime
  created_by: string

  updated_at: datetime
  updated_by: string
}
```

------

# 3. Конфигурация сущности

Каждый справочник задаётся конфигурацией:

```
{
  entity: "product-types" | "colors" | "clients" | "suppliers" | "sizes"

  title: "Типы товаров" | "Цвета" | "Клиенты" | "Поставщики" | "Размеры"

  api: "/product-types" | "/colors" | "/clients" | "/suppliers" | "/sizes"
}
```

------

# 4. Доступ

- только admin
- иначе 403

------

# 5. LIST PAGE

------

## 5.1 Маршрут

```
/dictionaries/{entity}
```

------

## 5.2 Структура (обязательно)

Используется:

- COMPONENT: PageContainer
- COMPONENT: Breadcrumbs
- COMPONENT: FiltersPanel
- COMPONENT: Table
- COMPONENT: Pagination
- SYSTEM: Query State Management

------

## 5.3 Breadcrumbs

```
Главная / Справочники / {title}
```

------

## 5.4 Таблица

Колонки:

1. Название
2. Актуален
3. Дата создания

------

### Сортировка

Используется:

- COMPONENT: Sorting

Доступна по:

- name
- is_active
- created_at

------

## 5.5 FiltersPanel

------

### Фильтры:

- Название (text, debounce)
- Актуален (Да / Нет)
- Дата создания

------

### Actions:

- CreateButton → `/dictionaries/{entity}/new`
- ResetFilters (внутри FiltersPanel)

------

## 5.6 Pagination

Используется:

- COMPONENT: Pagination

------

## 5.7 Поведение

------

### Инициализация:

1. читается Query State
2. GET /{entity}
3. отображается таблица

------

### Обновление:

Любое изменение:

- filters
- sort
- page
- limit

→ reload данных

------

### Row click:

```
/dictionaries/{entity}/{id}
```

------

# 6. CREATE PAGE

------

## 6.1 Маршрут

```
/dictionaries/{entity}/new
```

------

## 6.2 Структура

Используется:

- PageContainer (через компонент)
- Breadcrumbs (через компонент)
- Form (через компонент)
- ActionBar (через компонент)

------

## 6.3 Breadcrumbs

```
Главная / Справочники / {title} / Создание
```

------

## 6.4 Форма

Поля:

- Наименование (обязательное)
- Актуален (default: true)

------

## 6.5 Поведение

------

### Инициализация:

- форма пустая
- is_active = true

------

### Валидация:

- name обязателен

------

### Сохранение:

```
POST /{entity}
{
  "name": "string",
  "is_active": true
}
```

------

### Успех:

→ редирект на список
