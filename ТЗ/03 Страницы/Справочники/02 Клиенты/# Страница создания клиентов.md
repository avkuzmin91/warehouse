# СТРАНИЦА: СОЗДАНИЕ КЛИЕНТА

---

# 1. Цель

Реализовать страницу создания нового клиента справочника.

Страница предназначена для:
- создания нового клиента
- валидации введённых данных
- сохранения клиента в системе
- отправки данных в API

---

# 2. Зависимости

Использовать Data Model:
- Client

---

# 3. Доступ

- доступ только для роли **admin**
- при отсутствии доступа отображение 403 страницы

---

# 4. Маршрут

- /dictionaries/clients/new

---

# 5. UI структура страницы

---

## 5.1 Общая структура

Страница должна содержать (сверху вниз внутри карточки):

- хлебные крошки
- форму создания клиента
- кнопки управления

> Отдельный заголовок (h1) НЕ используется — название страницы задаётся через breadcrumbs.

---

## 5.2 Хлебные крошки

- реализуются через глобальный компонент Breadcrumbs
- формируются автоматически из URL `/dictionaries/clients/new`

Отображение: Главная / Справочники / Клиенты / Создание клиента

---

# 6. Форма

---

## 6.1 Общие правила

- поля соответствуют Data Model: Client
- обязательные поля помечены "*"
- форма вертикальная

---

## 6.2 Поля формы

### Название клиента (name)
- тип: текстовое поле
- обязательное
- отображать "*"

---

### Актуален (is_active)
- тип: checkbox
- по умолчанию: true

---

# 7. Поведение

---

## 7.1 Инициализация

- страница открывается по кнопке "Создать" со страницы:
  /dictionaries/clients
- форма пустая
- is_active = true

---

## 7.2 Валидация

При нажатии "Сохранить":

Проверяется:
- name заполнен

---

### Ошибки:

- сообщение: "Заполните обязательные поля"
- подсветка поля name

---

## 7.3 Кнопка "Сохранить"

- текст: "Сохранить"
- расположение: справа под формой
- hover-эффект

---

### Поведение при успехе:

- выполняется запрос: POST /clients

```json
{
  "name": "string",
  "is_active": true
}
```

- после успешного создания:
   → переход на /dictionaries/clients

------

### Ошибка API:

Пример:

- "Клиент уже существует"

Отображается пользователю без сброса формы

------

## 7.4 Кнопка "Отмена"

- текст: "Отмена"
- расположение: справа от "Сохранить"

------

### Поведение:

- переход на /dictionaries/clients
- данные не сохраняются

------

# 8. UI требования

- форма по центру экрана
- max-width контейнера
- адаптивность (mobile-friendly)
- отступы между элементами

------

# 9. Ограничения

- не использовать модальные окна
- не дублировать данные вне формы

------

# 10. Безопасность

- доступ только admin
- backend проверяет роль пользователя
- frontend не является источником прав доступа
- backend валидирует уникальность клиента (name)

------

# 11. Проверка

------

## Сценарий 1: успешное создание

1. Войти под admin
2. Открыть /dictionaries/clients
3. Нажать "Создать"
4. Заполнить поле name
5. Нажать "Сохранить"

Ожидается:

- клиент создан
- переход на список
- клиент отображается в таблице

------

## Сценарий 2: пустое поле

1. Нажать "Сохранить" без заполнения

Ожидается:

- сообщение "Заполните обязательные поля"
- подсветка поля

------

## Сценарий 3: ошибка backend

1. Ввести уже существующее имя клиента
2. Нажать "Сохранить"

Ожидается:

- ошибка "Клиент уже существует"

------

## Сценарий 4: отмена

1. Нажать "Отмена"

Ожидается:

- переход на список
- данные не сохраняются

------

## Проверка прав

1. зайти под user или manager
2. открыть /dictionaries/clients/new

Ожидается:

- 403 страница или редирект на /auth
