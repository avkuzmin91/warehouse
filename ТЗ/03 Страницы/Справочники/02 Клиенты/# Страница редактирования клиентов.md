# СТРАНИЦА: РЕДАКТИРОВАНИЕ КЛИЕНТА

---

# 1. Цель

Реализовать страницу редактирования клиента справочника.

Страница предназначена для:

- редактирования данных клиента
- сохранения изменений в системе
- отправки обновлённых данных в API

---

# 2. Зависимости

Использовать Data Model:

- Client

---

# 3. Доступ

- доступ только для роли **admin**
- при отсутствии доступа:
  - отображение 403 страницы

---

# 4. Маршрут

- /dictionaries/clients/{id}

---

# 5. UI структура страницы

---

## 5.1 Общая структура

Страница должна содержать:

- хлебные крошки
- форму редактирования
- кнопки управления

> Отдельный заголовок (h1) НЕ используется — название страницы задаётся через breadcrumbs.

---

## 5.2 Хлебные крошки

- реализуются через глобальный компонент Breadcrumbs
- формируются автоматически из URL

Для маршрута:
/dictionaries/clients/{id}

Отображение:

Главная / Справочники / Клиенты / Редактирование клиента

---

# 6. Форма

---

## 6.1 Общие правила

- поля соответствуют Data Model: Client
- форма заполняется данными из API
- обязательные поля помечены "*"

---

## 6.2 Загрузка данных

При открытии страницы:

- выполняется запрос:
  GET /clients/{id}

- результат заполняет форму

---

## 6.3 Поля формы

---

### Название клиента (name)

- тип: текстовое поле
- обязательное

---

### Актуален (is_active)

- checkbox
- значение берётся из backend

---

# 7. Поведение

---

## 7.1 Инициализация страницы

---

### Способ открытия страницы

Страница редактирования клиента открывается:

1. По клику на строку в списке клиентов

На странице:

- /dictionaries/clients

При нажатии на строку:
→ переход на страницу редактирования

---

### Маршрут

/dictionaries/clients/{id}

---

### Загрузка данных

После перехода:

- выполняется:
  GET /clients/{id}
- данные подставляются в форму

---

### Поведение при загрузке

- отображается loader
- форма disabled

---

### Ошибка загрузки

Если клиент не найден:

- показать:
  "Клиент не найден"
- кнопка "Назад" → /dictionaries/clients

---

## 7.2 Валидация

При нажатии "Сохранить":

Проверяется:

- name не пустой

---

### Ошибки:

- сообщение: "Заполните обязательные поля"
- подсветка поля name

---

## 7.3 Сохранение

Если валидация успешна:

- выполняется запрос:
  PATCH /clients/{id}

---

### Request:

```json
{
  "name": "string",
  "is_active": true
}
```

## 7.4 Успешное сохранение

- данные клиента обновляются
- редирект на:
   /dictionaries/clients

------

## 7.5 Ошибка API

Пример:

- "Клиент уже существует"

Поведение:

- показать сообщение
- форма не сбрасывается

------

# 8. Кнопки

------

## 8.1 Сохранить

- текст: "Сохранить"
- справа под формой

Поведение:

- валидация
- PATCH запрос

------

## 8.2 Отмена

- текст: "Отмена"
- справа от "Сохранить"

Поведение:

- переход на /dictionaries/clients
- без сохранения

------

# 9. UI требования

- форма по центру
- max-width
- адаптивность
- отступы между элементами

------

# 10. Ограничения

- не использовать модальные окна
- не использовать inline-редактирование

------

# 11. Безопасность

- доступ только admin
- backend проверяет:
  - роль
  - существование клиента
  - уникальность name

------

# 12. Служебные поля (Audit fields)

------

## 12.1 Общие правила

- только для отображения
- не редактируются
- приходят с backend

------

## 12.2 Поля

- Дата создания (created_at)

- Создал (created_by)
- Дата изменения (updated_at)
- Изменил (updated_by)

------

## 12.3 UI размещение

- внизу формы
- блок: "Информация о записи"

------

## 12.4 Пример

Информация о записи

Создано: 12.01.2026 10:45
 Создал: admin@system.com

Последнее изменение: 15.01.2026 14:20
 Изменил: manager@system.com

------

# 13. Проверка (Сценарии тестирования)

------

## Сценарий 1: Открытие

1. Войти под admin
2. Открыть /dictionaries/clients
3. Кликнуть на клиента

Ожидается:

- переход на /dictionaries/clients/{id}
- форма с данными

------

## Сценарий 2: Успешное редактирование

1. Изменить name
2. Нажать "Сохранить"

Ожидается:

- PATCH /clients/{id}
- редирект на список

------

## Сценарий 3: Пустое поле

1. Очистить name
2. Сохранить

Ожидается:

- ошибка
- подсветка

------

## Сценарий 4: Дубликат

1. Ввести существующее имя
2. Сохранить

Ожидается:

- "Клиент уже существует"

------

## Сценарий 5: Отмена

1. Нажать "Отмена"

Ожидается:

- возврат на список

------

## Сценарий 6: Нет доступа

1. Войти как user/manager
2. Открыть /dictionaries/clients/{id}

Ожидается:

- 403

------

## Сценарий 7: Не найден

1. Открыть /dictionaries/clients/invalid

Ожидается:

- "Клиент не найден"
