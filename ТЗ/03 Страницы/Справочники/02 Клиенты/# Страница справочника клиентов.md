# СТРАНИЦА: СПРАВОЧНИК КЛИЕНТОВ

---

# 1. Цель

Реализовать страницу списка клиентов справочника.

Страница предназначена для:
- просмотра клиентов
- перехода к карточке клиента
- перехода к созданию нового клиента

---

# 2. Доступ

- доступ только для роли **admin**
- при отсутствии доступа отображение 403 страницы

---

# 3. Маршрут

- /dictionaries/clients

---

# 4. UI структура страницы

---

## 4.1 Общая структура

Страница должна содержать (сверху вниз внутри карточки):

- хлебные крошки
- кнопку "Создать"
- таблицу клиентов
- пагинацию

> Заголовок страницы (h1) НЕ используется — название задаётся через breadcrumbs.

---

## 4.2 Хлебные крошки

- реализуются через глобальный компонент Breadcrumbs
- формируются автоматически из URL `/dictionaries/clients`

Отображение: Главная / Справочники / Клиенты

---

## 4.3 Таблица клиентов

Отображается в виде таблицы.

Колонки:

1. Название клиента
2. Статус (активен / неактивен)
3. Дата создания

---

## 4.4 Кнопка "Создать"

- текст: "Создать"
- расположение: над таблицей, справа
- поведение:
  → переход на /dictionaries/clients/new

---

## 4.5 Отображение данных

### Название клиента
- берётся из поля `name`

---

### Статус
- берётся из поля `is_active`
- отображение:
  - true → "Актуален"
  - false → "Не актуален"

---

### Дата создания
- формат: DD.MM.YYYY
- берётся из `created_at`

---

## 4.6 Строка таблицы

- каждая строка кликабельна
- hover-эффект обязателен
- курсор: pointer

---

## 4.7 Пагинация

- варианты: 20 / 50 / 100
- по умолчанию: 20
- расположение: справа под таблицей

---

# 5. Поведение

---

## 5.1 Загрузка данных

При открытии страницы: GET /clients

---

## 5.2 Переход в карточку клиента

При клике на строку:

→ переход на: /dictionaries/clients/{id}

---

## 5.3 Пустое состояние

Если клиентов нет:

- отображается пустая таблица без строк

---

## 5.4 Пагинация

- изменение limit:
  20 / 50 / 100

- page обновляется автоматически

---

# 6. UI требования

- таблица по центру экрана
- max-width контейнера
- адаптивность (mobile-friendly)
- горизонтальный скролл при необходимости
- отступы между элементами

---

# 7. Ограничения

- не использовать модальные окна
- не использовать inline-редактирование
- редактирование только на отдельной странице

---

# 8. API

---

## GET /clients

Параметры:

- page
- limit

---

# 9. Безопасность

- доступ только admin
- backend проверяет роль
- frontend не является источником прав доступа

---

# 10. Проверка

---

## Сценарий 1: загрузка списка

1. Войти под admin
2. Открыть /dictionaries/clients

Ожидается:
- отображается таблица клиентов

---

## Сценарий 2: создание клиента

1. Нажать "Создать"

Ожидается:
- переход на /dictionaries/clients/new

---

## Сценарий 3: открытие карточки

1. Нажать на клиента в таблице

Ожидается:
- переход на /dictionaries/clients/{id}

---

## Сценарий 4: пагинация

1. выбрать 50 элементов

Ожидается:
- количество строк изменилось

---

## Проверка прав

1. зайти под user или manager
2. открыть /dictionaries/clients

Ожидается:
- 403 страница или редирект на /auth
