# **COMPONENT: Date Range Filter**

------

# 1. Цель

Реализовать фильтр по диапазону дат:

- от (date_from)
- до (date_to)

Используется для фильтрации данных по периоду.

------

# 2. Область применения

Используется в:

- FiltersPanel
- list-страницах (таблицы)
- отчётах

------

# 3. UI

Фильтр состоит из двух полей:

1. Дата "с"
2. Дата "по"

------

## 3.1 Тип полей

- date picker
- формат отображения: `DD.MM.YYYY`

------

## 3.2 Placeholder

- "С даты"
- "По дату"

------

## 3.3 Расположение

- в одной строке
- логически объединены в один фильтр

------

# 4. Query State

Фильтр хранится в query params:

```
date_from=YYYY-MM-DD
date_to=YYYY-MM-DD
```

------

## Пример:

```
/products?date_from=2026-01-01&date_to=2026-01-31
```

------

# 5. Поведение

------

## 5.1 Изменение значений

При изменении:

- обновляется Query State
- выполняется API запрос

------

## 5.2 Частичное заполнение

Допускается:

- только date_from
- только date_to

------

### Логика:

- date_from → >= дата
- date_to → <= дата

------

## 5.3 Валидация

Если:

```
date_from > date_to
```

→ ошибка:

- сообщение: "Неверный диапазон дат"
- запрос НЕ отправляется

------

## 5.4 Очистка

При сбросе фильтров:

- date_from очищается
- date_to очищается

------

# 6. API

Передача в запросе:

```
{
  "date_from": "YYYY-MM-DD",
  "date_to": "YYYY-MM-DD"
}
```

------

# 7. UX правила

- значения сохраняются при перезагрузке (через Query State)
- при возврате на страницу фильтр восстанавливается
- debounce НЕ применяется (выбор через picker)

------

# 8. Ограничения

- нельзя вводить дату вручную (если используешь strict picker)
- timezone должен быть единый (обычно UTC или backend standard)

------

# 9. Итог

Date Range Filter — стандартный компонент фильтрации по периоду, полностью интегрированный с:

- Query State Management
- FiltersPanel
- API filtering
