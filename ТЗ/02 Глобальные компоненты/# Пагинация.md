# COMPONENT: Pagination

---

# 1. Цель

Реализовать универсальный компонент пагинации для списков данных.

Компонент используется на страницах:

- справочник товаров
- справочник клиентов
- другие списки системы

---

# 2. Зависимости

Использует:

- SYSTEM: Query State Management

---

# 3. Назначение

Компонент отвечает за:

- переключение страниц
- выбор количества элементов на странице
- синхронизацию состояния с URL
- триггер загрузки данных

---

# 4. UI структура

Компонент содержит:

- Переключение страниц (вперёд / назад)
- Номера страниц
- Выбор количества элементов на странице

---

## 4.1 Переключение страниц

Элементы:

- Кнопка "Назад"
- Кнопка "Вперёд"
- Номера страниц

---

### Поведение:

- текущая страница подсвечена
- кнопка "Назад" disabled на первой странице
- кнопка "Вперёд" disabled на последней странице

---

## 4.2 Выбор количества элементов

Dropdown:

- 20
- 50
- 100

По умолчанию:

- 20

---

# 5. Поведение

---

## 5.1 Изменение страницы

При выборе страницы:

- обновляется query param: page
- вызывается обновление данных

---

## 5.2 Изменение лимита

При изменении limit:

- обновляется query param: limit
- page сбрасывается в 1
- вызывается обновление данных

---

## 5.3 Синхронизация с URL

Состояние хранится в query params:

Пример:

/dictionaries/products?page=2&limit=50

---

## 5.4 Интеграция с Query State

Компонент НЕ хранит состояние локально.

Он:

- читает page, limit из Query State
- обновляет Query State при изменениях

---

# 6. API требования

Backend должен поддерживать:

GET /resource

Параметры:

- page (number)
- limit (number)

---

### Ответ:

```json
{
  "items": [],
  "total": 100,
  "page": 1,
  "limit": 20
}
```

# 7. UI требования

- размещается под таблицей
- выравнивание: справа
- отступ сверху от таблицы
- адаптивность

------

# 8. Ограничения

- не использовать локальное состояние вне Query State
- не дублировать логику пагинации на страницах

------

# 9. Проверка

------

## Сценарий 1: Переключение страницы

1. Нажать на страницу "2"

Ожидается:

- URL обновился: page=2
- загрузились новые данные

------

## Сценарий 2: Изменение limit

1. Выбрать 50 элементов

Ожидается:

- limit=50
- page=1
- данные обновились

------

## Сценарий 3: Граничные состояния

1. Перейти на первую страницу

Ожидается:

- кнопка "Назад" disabled

------

## Сценарий 4: URL синхронизация

1. Открыть:

/dictionaries/products?page=3&limit=100

Ожидается:

- отображается 3 страница
- лимит = 100
