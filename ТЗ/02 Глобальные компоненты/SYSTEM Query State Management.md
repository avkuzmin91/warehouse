# SYSTEM: QUERY STATE MANAGEMENT

---

# 1. Цель

Реализовать единый механизм управления состоянием списков данных.

Система предназначена для:

- фильтрации
- пагинации
- сортировки
- синхронизации состояния с URL
- формирования API-запросов

---

# 2. Область применения

Используется на страницах:

- /dictionaries/products
- /dictionaries/clients
- любые списочные страницы (таблицы)

---

# 3. Общая концепция

Все параметры списка управляются через единый объект состояния:

```ts
queryState = {
  page: number,
  limit: number,
  filters: {},
  sort: {}
}
```

# 4. Состав состояния

------

## 4.1 Пагинация

```typescript
page: number      // текущая страница
limit: number     // количество записей
```

4.2 Фильтры

```typescript
filters: {
  search?: string,
  type?: string,
  is_active?: boolean
}
```

4.3 Сортировка

```typescript
sort: {
  field?: string,
  direction?: "asc" | "desc"
}
```

# 5. Источник истины

------

## 5.1 Основной принцип

👉 URL = главный источник состояния

------

## 5.2 Пример

/dictionaries/products?page=1&limit=20&search=shirt&type=clothes&sort=created_at_desc

## 5.3 Правила

- при изменении UI → обновляется URL
- при открытии страницы → состояние берётся из URL
- React state синхронизируется с URL

------

# 6. Формирование API-запроса

------

## 6.1 Общий принцип

queryState → query params → API

------

## 6.2 Пример запроса

```http
GET /products?page=1&limit=20&search=shirt&type=clothes&sort=created_at_desc
```

## 6.3 Правила

- не отправлять пустые параметры
- boolean → true/false
- строки → без лишних пробелов

------

# 7. Поведение системы

------

## 7.1 Изменение фильтра

- обновляется filters
- page сбрасывается в 1
- обновляется URL
- выполняется GET запрос

------

## 7.2 Изменение страницы

- обновляется page
- обновляется URL
- выполняется GET запрос

------

## 7.3 Изменение лимита

- обновляется limit
- page сбрасывается в 1
- выполняется запрос

------

## 7.4 Изменение сортировки

- обновляется sort
- обновляется URL
- выполняется запрос

------

## 7.5 Сброс фильтров

- filters очищаются
- page = 1
- обновляется URL
- выполняется запрос

------

# 8. Архитектура компонентов

------

## 8.1 Страница

Отвечает за:

- вызов API
- передачу данных в таблицу
- интеграцию всех частей

------

## 8.2 Filters Panel

Отвечает за:

- изменение filters
- debounce поиска

------

## 8.3 Table

Отвечает за:

- отображение данных
- триггер сортировки

------

## 8.4 Pagination

Отвечает за:

- смену page
- смену limit

------

# 9. React реализация (рекомендация)

------

## 9.1 Хук

```typescript
useQueryState()
```

## 9.2 Возвращает

```typescript
{
  query,
  setFilters,
  setPage,
  setLimit,
  setSort,
  resetFilters
}
```

## 9.3 Поведение

- автоматически обновляет URL
- триггерит перезапрос данных

------

# 10. Синхронизация с URL

------

## 10.1 При загрузке страницы

- парсить query params
- инициализировать state

------

## 10.2 При изменении state

- обновлять URL через router

------

# 11. Ограничения

- не хранить состояние только в UI
- не дублировать state в нескольких местах
- не делать клиентскую фильтрацию вместо серверной

------

# 12. Безопасность

- backend валидирует:
  - page
  - limit
  - filters
- frontend не является источником истины

------
