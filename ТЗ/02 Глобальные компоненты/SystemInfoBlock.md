# # COMPONENT: SystemInfoBlock

------

# 1. Цель

Реализовать глобальный компонент для отображения служебной (audit) информации о записи.

Компонент предназначен для:

- отображения даты создания
- отображения пользователя, создавшего запись
- отображения даты последнего изменения
- отображения пользователя, изменившего запись

------

# 2. Область применения

Компонент используется на страницах:

- редактирования сущности (Edit)
- просмотра сущности (View)

 Не используется:

- на странице создания (Create)

------

# 3. Зависимости

- backend должен возвращать audit-поля
- используется внутри `PageContainer`
- используется совместно с:
  - Form
  - ActionBar

------

# 4. Контракт данных

Компонент принимает объект:

```
{
  "created_at": "string (ISO datetime)",
  "created_by": "string",
  "updated_at": "string | null",
  "updated_by": "string | null"
}
```

------

# 4.1 Типы данных (Type Definition)

Определяется единый интерфейс:

```
type SystemInfo = {
  created_at: string        // ISO datetime (обязательное)
  created_by: string        // string (email или имя) (обязательное)

  updated_at?: string | null
  updated_by?: string | null
}
```

## Правила типов

- `created_at` — всегда присутствует
- `created_by` — всегда присутствует
- `updated_at` — может отсутствовать
- `updated_by` — может отсутствовать

# 5. UI структура

## 5.1 Общая структура

```
SystemInfoBlock

  Заголовок: "Информация о записи"

  Блок 1:
    Создано: {created_at}
    Создал: {created_by}

  Блок 2:
    Последнее изменение: {updated_at}
    Изменил: {updated_by}
```

------

## 5.2 Пример отображения

```
Информация о записи

Создано: 12.01.2026 10:45
Создал: admin@system.com

Последнее изменение: 15.01.2026 14:20
Изменил: manager@system.com
```

------

# 6. Правила отображения

## 6.1 Формат даты

- формат: `DD.MM.YYYY HH:mm`
- локаль: системная

------

## 6.2 Если нет данных об изменении

Если запись не редактировалась:

```
Последнее изменение: —
Изменил: —
```

------

## 6.3 Обязательные поля

- created_at — обязательно
- created_by — обязательно

------

# 6.4 Преобразование данных (Data Mapping)

Перед отображением выполняется форматирование:

```
formatDate(created_at) → DD.MM.YYYY HH:mm
formatDate(updated_at) → DD.MM.YYYY HH:mm
```

---

# 7. Размещение на странице

Компонент размещается:

- внизу формы
- после всех редактируемых полей
- перед ActionBar

------

## Пример структуры страницы

```
Form Fields
↓
SystemInfoBlock
↓
ActionBar
```

------

# 7.1 Инициализация компонента

## Источник данных

Компонент получает данные:

- из API ответа:

```
GET /{entity}/{id}
```

------

## Момент инициализации

SystemInfoBlock инициализируется:

- **только после успешной загрузки данных сущности**
- вместе с формой редактирования / просмотра

---

# 7.2 Условия отображения

Компонент отображается, если:

```
created_at && created_by
```

------

## Если данных нет

Если backend не вернул обязательные поля: это считается ошибкой API

---

# 7.3 Логика отображения updated полей

```
if (!updated_at && !updated_by) {
  show("Последнее изменение: —")
  show("Изменил: —")
}
```

# 7.4 Обновление после сохранения

После успешного PATCH:

- данные audit должны обновиться
- компонент должен отобразить новые значения

------

## Источник обновления:

- либо повторный GET
- либо response PATCH

# 8. Поведение

- все поля только для чтения (read-only)
- не редактируются пользователем
- не участвуют в валидации формы
- не отправляются в API

------

# 9. Визуальные требования

- отображается как отдельный блок (section / card)
- визуально отделён от формы:
  - divider или фон
- меньший визуальный приоритет, чем форма

------

# 10. Состояния

## 10.1 Loading

Пока данные не загружены:

- компонент не отображается
   или
- отображается skeleton (опционально)

------

## 10.2 Error

Если audit-поля не пришли:

- отображается только доступная информация
- отсутствие updated_* не считается ошибкой

------

# 11. Безопасность

- данные приходят только с backend
- frontend не должен изменять значения
- backend отвечает за корректность данных

------

# 12. Ограничения

- запрещено:
  - редактировать audit-поля
  - использовать input-компоненты
  - размещать блок в середине формы
  - дублировать эти поля в других частях UI

------

# 13. Переиспользование

Компонент должен быть:

- универсальным для любых сущностей
- независимым от Data Model
- конфигурируемым только через props

------

# 14. Итог

`SystemInfoBlock` — это обязательный глобальный компонент для стандартизированного отображения audit-информации, обеспечивающий единообразие интерфейса и поведения на всех страницах редактирования и просмотра сущностей.
