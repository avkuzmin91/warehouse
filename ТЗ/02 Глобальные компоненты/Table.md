# COMPONENT: Table

---

# 1. Цель

Реализовать универсальный компонент таблицы для отображения списков данных.

Используется на страницах:

- справочник товаров
- справочник клиентов
- другие списки системы

---

# 2. Зависимости

Использует:

- COMPONENT: Sorting
- SYSTEM: Query State Management

---

# 3. Назначение

Компонент отвечает за:

- отображение списка данных
- рендер колонок
- обработку кликов по строкам
- интеграцию с сортировкой

---

# 4. Входные параметры (props)

Компонент принимает:

- columns — описание колонок
- data — массив данных
- loading — состояние загрузки
- onRowClick — обработчик клика по строке

---

## 4.1 columns

Массив объектов:

```ts
{
  key: string
  title: string
  sortable?: boolean
  render?: (value, row) => ReactNode
}
```

## 4.2 data

```typescript
Array<Record<string, any>>
```

------

## 4.3 loading

- boolean
- если true → отображается loader

## 4.4 onRowClick

- функция
- вызывается при клике на строку

------

# 5. UI структура

Таблица состоит из:

- Header (заголовки колонок)
- Body (строки данных)

------

## 5.1 Header

- отображает названия колонок
- поддерживает сортировку (если sortable=true)

------

## 5.2 Body

- отображает строки
- каждая строка:
  - кликабельна
  - имеет hover-эффект

------

# 6. Сортировка

Если колонка имеет:

```
sortable: true
```

→ используется:

- COMPONENT: Sorting

------

Поведение:

- при клике на заголовок:
  - обновляется Query State (sort)
  - происходит перезагрузка данных

------

# 7. Состояния

------

## 7.1 Loading

Если loading=true:

- отображается loader
- таблица неактивна

------

## 7.2 Empty state

Если data пустой:

- отображается строка:
   "Нет данных"

------

# 8. Поведение

------

## 8.1 Клик по строке

При клике:

- вызывается onRowClick(row)

------

## 8.2 Рендер значений

Если указан render:

```
render(value, row)
```

→ используется кастомный рендер

------

Примеры:

- изображение (mini preview)
- чекбокс
- форматированная дата

------

# 9. UI требования

- таблица по центру
- max-width
- горизонтальный скролл при переполнении
- выравнивание колонок
- отступы между строками

------

# 10. Ограничения

- не хранить состояние данных внутри компонента
- не выполнять API запросы
- не управлять фильтрацией

------

# 11. Пример использования

```
<Table
  columns={[
    { key: "name", title: "Название", sortable: true },
    { key: "sku", title: "Артикул", sortable: true },
    {
      key: "image",
      title: "Фото",
      render: (value) => <img src={value} width={40} />
    }
  ]}
```

# 12. Проверка

------

## Сценарий 1: Отображение данных

- передать data
- таблица отображает строки

------

## Сценарий 2: Клик по строке

- нажать на строку

Ожидается:

- вызывается onRowClick

------

## Сценарий 3: Сортировка

- нажать на заголовок sortable колонки

Ожидается:

- обновляется sort в Query State
- данные перезагружаются

------

## Сценарий 4: Пустое состояние

- data = []

Ожидается:

- "Нет данных"
