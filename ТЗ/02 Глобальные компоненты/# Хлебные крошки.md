# ГЛОБАЛЬНЫЙ КОМПОНЕНТ: ХЛЕБНЫЕ КРОШКИ (BREADCRUMBS)

---

# 1. Цель

Реализовать глобальный компонент хлебных крошек для навигации по приложению.

Компонент предназначен для:
- отображения текущего пути пользователя
- упрощения навигации между уровнями страниц
- стандартизации UI навигации

---

# 2. Область применения

Breadcrumbs отображаются на всех внутренних страницах приложения:

- /home
- /dictionaries/*
- /inventory/*
- /dictionaries/products/*
- /users/*
- другие защищённые страницы

---

# 3. Не отображается

Компонент НЕ отображается на страницах:

- /auth
- /auth/register

---

# 4. Структура компонента

Breadcrumbs представляет собой цепочку навигационных элементов:

Пример:
Главная / Справочники / Товары / Создание товара

---

# 5. Поведение

## 5.1 Формирование

- путь формируется автоматически на основе текущего URL
- каждый сегмент URL преобразуется в читаемое название

---

## 5.2 Навигация

- все элементы, кроме последнего, кликабельны
- последний элемент (текущая страница) не кликабелен

---

## 5.3 Пример

URL: /dictionaries/products/new

Отображение: Главная / Справочники / Товары / Создание товара

---

# 6. Правила отображения названий

## 6.1 Статические маппинги

- /home → Главная
- /dictionaries → Справочники
- /products → Товары
- /new → Создание
- /users → Пользователи

---

## 6.2 Динамические сегменты

- {id} → "Карточка"
- /edit → "Редактирование"

---

# 7. UI требования

- отображается в верхней части контента
- горизонтальное расположение
- разделитель: "/"
- последний элемент выделен (жирный или другой стиль)
- кликабельные элементы имеют hover-эффект

---

# 8. Техническая реализация (Frontend)

## 8.1 Компонент

- создать общий компонент Breadcrumbs
- использовать на всех layout-страницах

---

## 8.2 Пример использования

```jsx
<AppLayout>
  <Header />
  <Breadcrumbs />
  <Outlet />
</AppLayout>
