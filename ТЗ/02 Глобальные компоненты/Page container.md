# `Page Container` (UI Layer — только оболочка)

# 1. Цель

Определить единый UI-контейнер страниц приложения.

Отвечает только за:

- визуальную структуру страницы
- центрирование
- ограничение ширины
- единый стиль карточки

------

# 2. Область применения

Используется на всех страницах, кроме:

- `/auth`
- `/auth/register`

------

# 3. Концепция

PageContainer — это **визуальная оболочка**, внутри которой размещаются любые страницы:

- списки
- формы
- карточки
- настройки

------

# 4. Структура

```
<PageContainer>
  <Breadcrumbs />
  {children}
</PageContainer>
```

------

# 5. UI правила

## 5.1 Центрирование

- контейнер всегда по центру экрана
- horizontal center alignment

------

## 5.2 Ограничение ширины

- max-width задаётся извне (через страницу или паттерн)
- PageContainer НЕ принимает бизнес-решения

------

## 5.3 Внешний вид

- фон страницы — системный
- контейнер — card UI
- border-radius: 12–16px
- мягкая тень
- padding внутри контейнера

------

## 5.4 Адаптивность

- desktop: фиксированный max-width
- tablet: уменьшенные отступы
- mobile: full width + padding

------

# 6. Запрещено

PageContainer НЕ должен содержать:

- таблицы
- фильтры
- пагинацию
- кнопки действий
- бизнес-логику страниц

------

# 7. Ответственность

PageContainer отвечает только за:

- layout
- визуальный контейнер
- структуру страницы

------

# 8. Исключения

Auth страницы используют:

- `AuthLayout`

------

# 9. Итог

PageContainer = **чистый UI shell без логики**
