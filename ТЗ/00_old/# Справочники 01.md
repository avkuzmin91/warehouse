\# DICTIONARIES MODULE ТЗ (СПРАВОЧНИКИ)

\---

\# 1. Цель
Реализовать систему справочников:

\- Клиенты
\- Цвета
\- Размеры
\- Товары

Все справочники:
\- управляются только администратором
\- используются в других модулях (товары, склад, заказы)

\---

\# 2. Общие правила для всех справочников

\## Поля (базовые)
\- id: UUID
\- name: string (обязательное)
\- is_active: boolean (по умолчанию true)
\- created_at: datetime

\---

\## Бизнес-правила
\- поле name обязательно
\- поле name должно быть уникальным (в рамках справочника)
\- is_active = false означает "Не актуален"
\- записи не удаляются без подтверждения

\---

\## Доступ
\- CRUD доступ только для роли admin
\- user и manager не имеют доступа

\---

\## Удаление
При удалении:
\- frontend обязан показать подтверждение:

Текст:
"Данные в карточках могут быть утеряны"

Кнопки:
\- Да (удалить)
\- Нет (отмена)

\---

\# 3. Backend (FastAPI)

\---

\## 3.1 RBAC
Все endpoints справочников:
\- только для admin

\---

\## 3.2 API (унифицированный подход)

Для каждого справочника:

\- GET /{dictionary}
\- POST /{dictionary}
\- PATCH /{dictionary}/{id}
\- DELETE /{dictionary}/{id}

\# 7. Справочник товаров

\## Entity: Product

Поля:
\- id
\- name (required)
\- type: enum ("clothes", "electronics")
\- sku (string, required, unique)
\- supplier (string, optional)
\- image_url (string, optional)
\- is_active (boolean)
\- created_at (datetime)

\---

\## Дополнительно (на будущее)
Если type = clothes:
\- можно будет связать с:
  \- colors
  \- sizes

\---

\## API

\- GET /products
\- POST /products
\- PATCH /products/{id}
\- DELETE /products/{id}

\---

\## 9.6 Доступ

\- если не admin → редирект на /home

\---

\# 10. Бизнес-правила (важно)

\- нельзя создать запись с пустым name
\- нельзя создать дубликат name
\- нельзя создать товар с дублирующимся sku
\- удаление требует подтверждения
\- is_active используется вместо физического удаления (опционально можно заменить DELETE на soft delete)

\---

\# 11. Проверка

\## Сценарий:

1. Зайти под admin
2. Перейти в "Справочники"
3. Создать:

   \- клиента
   \- цвет
   \- размер
   \- товар

4. Отредактировать запись
5. Удалить запись → проверить подтверждение


\---

\## Проверка прав:

\- зайти под user/manager
\- попытаться открыть /dictionaries

Ожидается:
\- отказ доступа или редирект
