# # DATA MODEL: PRODUCT

---

# 1. Общая информация

Сущность Product (Товар) используется для:
- хранения информации о товаре
- использования в складских операциях (поступления, отгрузки)
- отображения в справочнике товаров

---

# 2. Entity: Product

## Поля

### id
- тип: UUID
- первичный ключ

---

### name (Название товара)
- тип: string
- обязательное
- не уникальное

---

### type (Тип товара)
- тип: enum
- обязательное

Значения:
- Одежда
- Техника

---

### sku (Артикул товара)
- тип: string
- обязательное
- уникальное

---

### supplier (Поставщик)
- тип: string
- необязательное

---

### image_url (Фото товара)
- тип: string
- хранит путь или URL к изображению
- необязательное

---

### is_active (Актуален)
- тип: boolean

Значения:
- true → товар актуален
- false → товар не актуален

По умолчанию:
- true

---

### created_at (Дата создания)
- тип: datetime
- устанавливается автоматически

---

### updated_at (Дата обновления)
- тип: datetime
- обновляется при изменении

---

# 3. Бизнес-правила

- sku (артикул) должен быть уникальным
- name не может быть пустым
- type обязателен
- нельзя создать товар без sku
- image_url хранит только ссылку, не бинарные данные

---

# 4. Связи (на будущее)

## 4.1 Product ↔ Stock (склад)
- один товар участвует во множестве операций:
  - поступления
  - отгрузки

---

# 5. Ограничения

- sku уникален на уровне базы данных
- удаление товара:
  - возможно только если нет связанных операций (опционально)
  - либо использовать soft delete через is_active

---

# 6. Пример JSON

```json
{
  "id": "uuid",
  "name": "Футболка базовая",
  "type": "clothes",
  "sku": "TSHIRT-001",
  "supplier": "ООО Поставщик",
  "image_url": "/uploads/tshirt.jpg",
  "is_active": true,
  "created_at": "2026-01-01T12:00:00",
  "updated_at": "2026-01-01T12:00:00"
}
```
