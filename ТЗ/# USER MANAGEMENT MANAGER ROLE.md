# USER MANAGEMENT: ADMIN + MANAGER ROLE

---

# 1. Цель
Добавить:
- роль **manager**
- возможность администратору:
  - назначать роль manager пользователям
  - удалять пользователей

---

# 2. Роли системы

## user (default)
- регистрация и вход
- доступ к /dashboard
- нет доступа к управлению пользователями

---

## manager
- вход в систему
- доступ к функционалу склада (будет добавлен позже)
- не имеет доступа к управлению пользователями

---

## admin
- полный доступ
- управление пользователями
- назначение ролей
- удаление пользователей

---

# 3. Data Model (обновление)

## User

Поля:
- id: UUID
- email: string (unique)
- password_hash: string
- role: string ("user" | "manager" | "admin")
- created_at: datetime

---

# 4. Backend (FastAPI)

---

## 4.1 Обновление JWT

JWT должен содержать:
- user_id
- email
- role

---

## 4.2 RBAC (role-based access control)

Создать зависимости:

- get_current_user (уже есть)
- get_current_admin
- get_current_manager (на будущее)

### get_current_admin:
- если role != "admin" → вернуть 403

---

## 4.3 API Endpoints

---

### GET /users
Получить список всех пользователей

Доступ:
- только admin

Response:
```json
[
  {
    "id": "uuid",
    "email": "string",
    "role": "user | manager | admin",
    "created_at": "datetime"
  }
]
```

------

### PATCH /users/{id}/role

Назначение роли пользователю

Доступ:

- только admin

Request:

```
{
  "role": "manager"
}
```

Правила:

- можно назначить только "manager" или "user"
- нельзя назначить "admin" через API
- нельзя изменить роль самому себе (опционально, но рекомендуется)

Response:

```
{
  "message": "Role updated"
}
```

------

### DELETE /users/{id}

Удаление пользователя

Доступ:

- только admin

Правила:

- нельзя удалить самого себя
- пользователь полностью удаляется из базы

Response:

```
{
  "message": "User deleted"
}
```

------

# 5. Бизнес-правила

- только admin управляет пользователями
- manager не имеет доступа к /users
- user не имеет доступа к /users
- роли валидируются на backend
- password_hash никогда не возвращается

# 9. Проверка

## Сценарий:

1. Залогиниться как admin
2. Перейти на /users
3. Убедиться, что отображаются все пользователи

------

## Проверка ролей:

- назначить пользователю роль manager
- перелогиниться под этим пользователем
- убедиться, что доступ к /users отсутствует

------

## Проверка удаления:

- нажать удалить пользователя
- проверить появление модального окна
- подтвердить удаление
- пользователь исчезает из списка

------

# 10. Нефункциональные требования

- все операции защищены RBAC
- frontend не является источником безопасности
- backend проверяет роль на каждом запросе
- UI должен корректно обрабатывать ошибки API
