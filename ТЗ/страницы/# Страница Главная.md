# # СТРАНИЦА: ГЛАВНАЯ

---

# 1. Цель
Реализовать главную страницу приложения.

Страница предназначена для:
\- навигации по основным модулям системы
\- отображения доступных разделов в зависимости от роли пользователя

---

# 2. Доступ

- доступ к странице для всех авторизованных пользователей
- для неавторизованных пользователей: → редирект на /auth

---

# 3. Маршрут

/home

---

# 4. UI структура страницы

## 4.1 Общая структура

Страница должна содержать:

- Заголовок
- Список навигационных элементов

---

## 4.2 Заголовок

Текст: "Главная"

---

## 4.3 Список навигации

Отображается в виде списка

---

### Элементы

1. Справочники
   - текст: "Справочники"
   - доступ: только admin
   - переход: /dictionaries
2. Учет товаров
   - текст: "Учет товаров"
   - доступ: admin, manager
   - переход: /inventory (заглушка)

---

## 4.4 Отображение по ролям

- admin: видит все пункты
- manager: видит только "Учет товаров"
- user: не видит ни одного пункта

## 4.5 Пустое состояние

Если нет доступных разделов, то отобразить сообщение: "Нет доступных разделов"

---

## 5. Поведение

## 5.1 Загрузка страницы

- определяется роль пользователя
- отображаются только доступные разделы

---

## 5.2 Переходы

При клике:

- Справочники → /dictionaries
- Учет товаров → /inventory

---

# 6. UI требования

- элементы расположены по центру экрана
- ограничение ширины (max-width)
- вертикальный список
- адаптивность (mobile-friendly)
- hover-эффект для элементов
- отступы между элементами

---

# 7. Ограничения

- не использовать таблицы
- не использовать модальные окна
- страница выполняет только навигационную функцию

# 8. Безопасность

- backend проверяет авторизацию
- frontend не является источником прав доступа
  - при попытке доступа к недоступному разделу: → backend возвращает 403

---
