# # СТРАНИЦА СПРАВОЧНИКА ТОВАРОВ

---

# 1. Цель
Реализовать страницу списка товаров справочника.

Страница предназначена для:

- просмотра товаров
- перехода к карточке товара
- перехода к созданию нового товара

---



# 2. Доступ

- доступ только для роли **admin**
- при отсутствии доступа отображение 403 страницы

---

# 3. Маршрут

/dictionaries/products

---

# 4. UI структура страницы

## 4.1 Общая структура

Страница должна содержать (сверху вниз внутри карточки):

- Хлебные крошки (см. п. 4.2) — **название экрана** задаётся крошками (последний сегмент: **«Товары»**); отдельный заголовок страницы (h1) **не** выводится
- Кнопку "Создать"
- Таблицу товаров
- Пагинация

---

## 4.2 Хлебные крошки

- Реализуются **общим компонентом** `Breadcrumbs` (см. ТЗ `02 Глобальные компоненты / # Хлебные крошки.md`): путь **автоматически** строится по текущему URL (`/dictionaries/products`), без дублирования логики только на этой странице.
- **Размещение:** первый блок **внутри** стеклянной карточки (`auth-card`), **над** кнопкой «Создать» и таблицей.
- **Внешний вид:** разделитель «/» с **пробелами с обеих сторон**; цвета и отступы — по глобальному ТЗ о крошках.
- **Текстовая цепочка** для маршрута `/dictionaries/products` (последний сегмент — текущая страница, не ссылка):

`Главная / Справочники / Товары`

> **Примечание:** сегмент URL `products` в крошках отображается как **«Товары»**; дублирующий текстовый заголовок под крошками **не** требуется.

---

## 4.3 Таблица товаров

Отображается в виде таблицы

Колонки:

1. Название товара
2. Тип
3. Артикул товара
4. Поставщик
5. Фото товара
6. Актуален
7. Дата создания

---

## 4.4 Кнопка "Создать"

- текст: "Создать"
- расположение: над таблицей, справа
- поведение:  → переход на /dictionaries/products/new

## 4.5 Отображение данных

- Название, тип, артикул, поставщик, актуален - берутся значения из соответствующих полей отображаемого элемента справочника
- Фото:
    \- отображается миниатюра изображения, если добавлена в карточке товара
    \- фиксированный размер 40x40

- Актуален: чекбокс "Да / Нет"
- Дата создания: формат: DD.MM.YYYY

## 4.6 Строка таблицы

Каждая строка:

- кликабельна
- имеет hover-эффект

---

## 4.7 Пагинация

- Варианты: 20 / 50 / 100 элементов
- По умолчанию - 20
- Выбор количества отображаемых записей
- Расположение: под таблицей, справа

---

## 5. Поведение

## 5.1 Загрузка данных

При открытии страницы:

1. выполняется GET /products
2. отображается список товаров

---

## 5.2 Переходы

При клике на строку:
→ переход на: /dictionaries/products/{id}

---

## 5.3 Пагинация

При выборе количество строк меняется на 20 / 50 / 100 соответственно

---

## 5.4 Пустое состояние

Если товаров нет:
\- отображать таблицу без строк

---

# 6. UI требования

- таблица расположена по центру экрана
- ограничение ширины (max-width)
- адаптивность (mobile-friendly)
- горизонтальный скролл при переполнении
- отступы между элементами

---

# 7. Ограничения

- не использовать модальные окна
- не использовать inline-редактирование
- редактирование только на отдельной странице



## 8. API

- GET /products (с поддержкой пагинации)
- параметры:

  	\- page
 	
 	 \- limit

---

# 9. Безопасность

- доступ к API только для admin
- backend проверяет роль пользователя
- frontend не является источником прав доступа

---

# 10. Проверка

## Сценарий:

1. Войти под admin
2. Перейти на /dictionaries/products
3. Убедиться, что отображается таблица товаров
4. Нажать "Создать" → переход на форму
5. Нажать на товар → переход в карточку

---

## Проверка прав:

1. зайти под user или manager
2. открыть /dictionaries/products
3. Ожидается ошибка 403
