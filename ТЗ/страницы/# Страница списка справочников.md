# # СТРАНИЦА СПРАВОЧНИКИ

---

# 1. Цель
Реализовать страницу со списком справочников системы.

- Пользователи
- Клиенты
- Цвета
- Размеры
- Товары

Страница является точкой входа в модуль справочников.

---



# 2. Доступ

- доступ только для роли **admin**
- при отсутствии доступа отображение 403 страницы

---



# 3. Маршрут

/dictionaries

---



# 4. UI структура страницы

## 4.1 Общая структура

Страница должна содержать:

- Хлебные крошки
- Заголовок страницы
- Список справочников
- Кнопку "Назад"

---

## 4.2 Хлебные крошки

Формат:
Главная / Справочники

---

## 4.3 Заголовок

Текст:
"Справочники"

---

## 4.4 Список справочников

Отображается в виде списка

Порядок:

1. Пользователи
2. Товары
3. Клиенты
4. Размеры
5. Цвета

---

### 4.5 Элемент списка

Каждый элемент должен содержать:
\- название справочника

Каждый элемент:
\- визуально кликабельный
\- имеет hover-эффект

---



## 5. Поведение

## 5.1 Переходы

При клике на элемент:

- Пользователи → /users

- Товары → /dictionaries/products
- Клиенты → /dictionaries/clients
- Размеры → /dictionaries/sizes
- Цвета → /dictionaries/colors

## 5.2 Кнопка "Назад"

\- возвращает на предыдущую страницу (history.back)

---

# 6. UI требования

- список справочников должен быть расположен по центру экрана

- ограничение ширины (max-width)
- адаптивность (mobile-friendly)
- выравнивание элементов по вертикали
- достаточные отступы между элементами

---

# 7. Ограничения

- не использовать таблицы
- не использовать модальные окна
- не смешивать с содержимым справочников
- страница выполняет только навигационную функцию

# 8. Безопасность

- доступ к странице только через проверку роли на backend
- frontend не является источником прав доступа

---

# 9. Проверка

## Сценарий:

1. Войти под admin
2. Перейти на /dictionaries
3. Убедиться, что отображается список справочников
4. Нажать на любой справочник
5. Убедиться, что происходит переход

---

## Проверка прав:

1. зайти под user или manager
2. открыть /dictionaries
3. ошибка 403
